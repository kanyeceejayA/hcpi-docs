    Linux Server Installation

    This guide walks you through installing HCPI on a Linux server (Ubuntu 20.04 LTS or later recommended).

    Step 1: Update System Packages

    sudo apt update
sudo apt upgrade -y

    Step 2: Install System Dependencies

    Install required packages:

    sudo apt install -y git python3 python3-pip python3-dev python3-venv \
    postgresql postgresql-contrib libpq-dev \
    build-essential libssl-dev libffi-dev libxml2-dev libxslt1-dev \
    zlib1g-dev libjpeg-dev libsasl2-dev libldap2-dev \
    node-less npm wkhtmltopdf

    Step 3: Configure PostgreSQL

    Create a PostgreSQL user for Odoo:

    sudo -u postgres createuser -s $USER

    Create a database (you'll configure this in Odoo):

    createdb hcpi

    Database Password

    For production systems, set a password for the PostgreSQL user: 

    sudo -u postgres psql -c "ALTER USER $USER WITH PASSWORD 'your_secure_password';"

    Step 4: Create Directory Structure

    Create the HCPI directory structure:

    sudo mkdir -p /opt/hcpi
sudo chown $USER:$USER /opt/hcpi
cd /opt/hcpi

    Step 5: Download and Extract HCPI Files

    Download the files from http://statistics.ubos.org/hcpishare to your local machine, then transfer them to the server, or use wget/curl if accessible:

    # If you've transferred the file to your server
cd /opt/hcpi
unzip hcpi-code.zip

    This will extract the following folders: - conf/ - Configuration files - custom/ - Contains the HCPI module - log/ - Log files - odoo/ - Odoo codebase - venv/ - Python virtual environment

    Step 6: Set Up Python Virtual Environment

    Create and activate the virtual environment:

    cd /opt/hcpi
python3 -m venv venv
source venv/bin/activate

    Install Python dependencies:

    pip install --upgrade pip
pip install wheel
pip install -r odoo/requirements.txt

    Step 7: Configure Odoo

    Create or edit the configuration file:

    nano /opt/hcpi/conf/odoo.conf

    Add the following configuration (adjust as needed):

    [options]
admin_passwd = admin_password_change_this
db_host = localhost
db_port = 5432
db_user = your_username
db_password = your_db_password
addons_path = /opt/hcpi/odoo/addons,/opt/hcpi/custom
logfile = /opt/hcpi/log/odoo.log
log_level = info

    Security

    Change admin_passwd to a strong password. This is used for database management operations.

    Step 8: Restore Database (Optional)

    If you want to start with sample data, restore the database:

    unzip hcpi-sql.zip
psql hcpi < hcpi_dump.sql

    Skip for Empty Instance

    Skip this step if you want to start with an empty HCPI instance. Odoo will initialize the database when you first run it.

    Step 9: Start HCPI

    Start Odoo with the HCPI configuration:

    cd /opt/hcpi
source venv/bin/activate
python odoo/odoo-bin -c conf/odoo.conf

    First Run with Empty Database

    If you didn't restore a database, add the -i HCPI flag to install the HCPI module on first run: 

    python odoo/odoo-bin -c conf/odoo.conf -i HCPI

    Step 10: Access HCPI

    Open your web browser and navigate to:

    http://localhost:8069

    Or if accessing remotely:

    http://your_server_ip:8069

    Running as a Service (Recommended for Production)

    Create a systemd service file for automatic startup:

    sudo nano /etc/systemd/system/hcpi.service

    Add the following content:

    [Unit]
Description=HCPI Odoo Service
After=network.target postgresql.service

[Service]
Type=simple
User=your_username
Group=your_username
WorkingDirectory=/opt/hcpi
Environment="PATH=/opt/hcpi/venv/bin"
ExecStart=/opt/hcpi/venv/bin/python /opt/hcpi/odoo/odoo-bin -c /opt/hcpi/conf/odoo.conf
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target

    Enable and start the service:

    sudo systemctl daemon-reload
sudo systemctl enable hcpi
sudo systemctl start hcpi

    Check service status:

    sudo systemctl status hcpi

    Troubleshooting

    Check Logs

    View Odoo logs:

    tail -f /opt/hcpi/log/odoo.log

    Service Issues

    If the service fails to start:

    sudo systemctl status hcpi
sudo journalctl -u hcpi -n 50

    Port Already in Use

    If port 8069 is already in use, change it in the config file:

    xmlrpc_port = 8070
